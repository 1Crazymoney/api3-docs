A package/CLI tool to interact with Airnode across blockchains. There are commands for both requesters and API providers. Requesters can endorse client contracts and fund Airnodes. API providers can build Airnodes that serve their API data to requester client contracts.

Almost all commands require you to provide a providerUrl such as https://ropsten.infura.io/v3/<KEY>, https://xdai.poanetwork.dev, etc. The CLI connects to AirnodeRrp.sol (opens new window) contract, which address is derived from the current chain. You can optionally specify the contract address yourself by providing optional airnodeRrp command argument with address of the deployed contract on your targeted chain.

Commands that require mnemonic will make an on-chain transaction. The application will derive the account from the mnemonic with default ethereum derivation path m/44'/60'/0'/0/0, but you can override this by derivationPath flag. Make sure that the wallet that is associated with the mnemonic is funded on the target chain. The application will not exit until the transaction is confirmed.

View all commands:

npx @api3/airnode-admin --help
1

View the parameters of a command:

npx @api3/airnode-admin $COMMAND --help
1

# SDK

You can also use the package programmatically. The SDK exports respective functions for all CLI commands as well as helper functions for obtaining the contract instance on the targeted chain.

import { createRequester, getAirnodeRrpWithSigner } from '@api3/admin';

// First obtain the contract instance on target chain
const airnodeRrp = await getAirnodeRrpWithSigner(mnemonic, derivationPath, providerUrl, airnodeRrpAddress);
// Pass the contract instance as the first argument to the SDK function
const requesterIndex = await createRequester(airnodeRrp, requesterAdmin);

If you plan to use multiple commands it might be tedious to pass the contract instance to every function call. For this reason there is also class based AdminSdk which you initialize with AirnodeRrp contract only once.

import { AdminSdk } from '@api3/admin';

// First initialize the SDK with AirnodeRrp contract instance.
// You can use static AdminSdk functions or provide your own instance.
const airnodeRrp = await AdminSdk.getAirnodeRrpWithSigner(mnemonic, derivationPath, providerUrl, airnodeRrpAddress);
// Create sdk instance
const adminSdk = new AdminSdk(airnodeRrp);
// Call the method you need
const requesterIndex = await adminSdk.createRequester(requesterAdmin);

// You can switch the contract instance anytime. E.g. if you are using ethers
adminSdk.airnodeRrp = airnodeRrp.connect(someOtherWallet);

# Requester commands

# create-requester

Creates a requester (opens new window) and returns a requester index. Note down your requester index because you will be using it in future interactions.

npx @api3/airnode-admin create-requester \
  --providerUrl https://ropsten.infura.io/v3/<KEY> \
  --mnemonic "nature about salad..." \
  --requesterAdmin 0x5c17cb...

# set-requester-admin

Sets the requester admin (opens new window). The account derived from the mnemonic you provide here has to belong to the previous requester admin.

npx @api3/airnode-admin set-requester-admin \
  --providerUrl https://ropsten.infura.io/v3/<KEY> \
  --mnemonic "nature about salad..." \
  --requesterIndex 6 \
  --requesterAdmin 0xe97301...

# endorse-client

Endorses (opens new window) a client contract so that its requests can be fulfilled by the requester's designated wallet. The account derived from the mnemonic you provide here has to belong to the requester admin.

npx @api3/airnode-admin endorse-client \
  --providerUrl https://ropsten.infura.io/v3/<KEY> \
  --mnemonic "nature about salad..." \
  --requesterIndex 6 \
  --clientAddress 0x2c2e12...

# create-template

Reads a file, uses its contents to create a template (opens new window) and returns the template ID. See the /example directory for an example template file.

npx @api3/airnode-admin create-template \
  --providerUrl https://ropsten.infura.io/v3/<KEY> \
  --mnemonic "nature about salad..." \
  --templateFilePath ./template.json

# request-withdrawal

Requests a withdrawal (opens new window) from the wallet designated by an Airnode for a requester, and returns the request ID. The account derived from the mnemonic you provide here has to belong to the requester admin.

npx @api3/airnode-admin request-withdrawal \
  --providerUrl https://ropsten.infura.io/v3/<KEY> \
  --mnemonic "nature about salad..." \
  --airnodeId 0xe1e0dd... \
  --requesterIndex 6 \
  --destination 0x98aaba...

# Airnode commands

# set-airnode-parameters

Sets the parameters of an Airnode (opens new window) and returns the Airnode ID. See the /example directory for an example authorizers file.

You probably should not be using this. Airnode will set its own parameters during deployment (opens new window) if necessary.

npx @api3/airnode-admin set-airnode-parameters \
  --providerUrl https://ropsten.infura.io/v3/<KEY> \
  --mnemonic "nature about salad..." \
  --airnodeAdmin 0xc2193d... \
  --authorizersFilePath ./authorizers.json
